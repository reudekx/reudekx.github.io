# 블로그

## 개요

블로그 개발과 관련된 문서들입니다.

## 개발 예정 사항

* a 태그 관련
    * 타이틀 속성 추가하기 (JS 대신 파싱하여 빌드 시 삽입)
    * 외부 링크는 새 창에서 열리도록 하기 (JS 대신 파싱하여 빌드 시 삽입)  
    `<a href="https://example.com" target="_blank" rel="noopener noreferrer">`

* ~~문서 카테고리 추가하기~~ (완료)

* [검색 기능 추가하기](/docs/블로그/검색 기능 추가하기)

* [메인](/) 페이지에 최근 수정된 문서 목록 표시하기

* [루트](/docs) 문서에 어떤 정보 나타낼지 기획하기

* 디렉터리 역할만 수행하는 문서들이 있는데, 해당 페이지들을 위한 레이아웃을 따로 만들어야 할지 고민 중
    * 가령 하위 문서 목록을 나타낼 수 있는 기능을 추가
        * 기존 마크다운 문법과 충돌하지 않도록 기호를 몇개 골라 커스텀 문법을 만들어서 넣어도 될 듯
            * 현재는 parse_html 옵션을 꺼놨는데, liquid 문법은 여전히 사용 가능하긴 함.
            * 물론 parse_html 옵션을 다시 켜도 됨. (혹시 후처리를 할 때 문제가 생길까봐 꺼놓은 것인데
            다시 키더라도 웬만한 경우에선 문제가 이뤄지지 않을 것 같다.
            가령 인라인 코드나 코드 블록 내부의 html 태그는 현재도 이스케이프 처리가 자동으로 이뤄지고 있음.)

* 문서 트리 구조를 접을 수 있도록 하기 (혹은 페이지의 가독성을 해치지 않울 수 있도록 다른 방법 도입)

* 다른 문서를 링크할 때 편의 문법 제공하기
    * 예를 들어 마크다운에서 url을 `$알고리즘`과 같이 적으면 문서 트리 구조상 가장 먼저 식별되는 문서의 링크로 대체되도록 한다.
    * 만약 커스텀 문법을 정의한다면 이 기능도 포함하여 일괄적으로 처리하면 좋을 듯.
    * 또한 문서의 목차도 텍스트로 식별할 수 있도록 목차를 `<section-title>` 같은 식으로 감싼 뒤
        `$알고리즘/이분 탐색.매개변수 탐색` 이렇게 참조할 수 있도록 해도 된다.
