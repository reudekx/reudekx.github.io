# 날짜 자동으로 추가하기

## 개요

문서의 생성 및 수정 날짜를 자동적으로 추가해 보자.

## Git Hooks

Git Hooks를 이용하면 어떠한 Git 이벤트가 발생했을 때
특정 스크립트가 실행되도록 할 수 있다.
이를 이용하여 커밋 시점에 날짜가 자동으로 마크다운 파일에 삽입되도록 해 보자.

다만 현재 Github Pages의 기본 배포 파이프라인인
`github-pages`를 이용하고 있기에
Git Hooks를 사용할 수 있을지 잠시 염려가 되었지만,
내가 사용할 `Committing-Workflow` 훅의 경우 로컬에서 실행되기 때문에 상관이 없었다.

## 구현

`pre-commit` 훅을 이용하여 다음 기능을 구현할 것이다.
요구 사항은 다음과 같다.

* 새로 생성된 문서의 경우 created_date를 front matter에 추가한다.
* 수정된 문서의 경우 modified_date를 front matter에 추가하거나 갱신한다.
* created_date가 이미 있는 경우에는 실제 생성 날짜와 달라도 갱신을 하지 않는다.
* 있어야 할 created_date와 modified_date가 없는 경우 적절히 추가한다.
* 시간까지 나타내지는 않고 날짜만 추가한다.

이제 구현을 시작해 보자.

### 경로 변경 및 파일 생성

먼저 훅 파일을 깃 저장소에 업로드할 수 있도록
`.git/hooks`로 지정된 기본 훅 경로를 변경하자.

```bash
mkdir .githooks
git config core.hooksPath .githooks
```

다음으로는 pre-commit 파일을 생성한 뒤 실행 권한을 부여하자.

```bash
touch .githooks/pre-commit
chmod +x .githooks/pre-commit
```

### pre-commit 파일 작성

다음과 같은 셸 스크립트를 pre-commit 파일에 작성하자.

```bash
#!/bin/bash

CURRENT_DATE=$(date +%Y-%m-%d)

process_file() {
    local file="$1"

    # 첫 줄이 정확히 "---"로 시작하는지 검사
    if ! head -n1 "$file" | grep -q "^---$"; then
        return 0
    fi

    # 임시 파일 생성
    if ! tmp_file=$(mktemp); then
        echo "Error: Failed to create temporary file"
        return 1
    fi

    # 첫 줄을 임시 파일에 출력
    head -n1 "$file" > "$tmp_file"

    # 생성 날짜가 없으면 임시 파일에 출력
    if ! grep -q "^created_date:" "$file"; then
        echo "created_date: $CURRENT_DATE" >> "$tmp_file"
    fi

    # 수정 날짜 갱신
    # sed 's/패턴/바꿀내용/횟수' (s = substitution)
    tail -n +2 "$file" | sed "s/^modified_date:.*/modified_date: $CURRENT_DATE/1" >> "$tmp_file"

    # 수정 날짜가 없는 경우 임시 파일을 수정하여 추가
    # 첫번째 created_date 이후는 무시하고,
    # 첫번째 created_date 발견시에는 다음 줄에 modified_date 추가
    # 여러 옵션을 시도하였고 아래의 명령어만이 잘 작동했다...
    if ! grep -q "^modified_date:" "$tmp_file"; then
        sed -i "0,/^created_date:/!b;/^created_date:/a modified_date: $CURRENT_DATE" "$tmp_file"
    fi

    mv "$tmp_file" "$file"

    git add "$file"
}

# 스테이징된 마크다운 파일 목록
staged_files=$(git diff --cached --name-only --diff-filter=d | grep -E '\.md$|\.markdown$')

# 파일명에 공백이 있을 수 있으므로
# IFS를 빈 값으로 설정하고 한 줄씩 읽어 처리
echo "$staged_files" | while IFS= read -r file; do
    # 파일이 존재하는 경우 처리
    if [[ -f "$file" ]]; then
        process_file "$file"
        echo "Updated dates (created/modified) for: $file"
    fi
done

exit 0
```
### 문제 해결

파일 경로에 영어 이외의 문자가 있는 마크다운 파일의 경우
`grep` 명령어를 통해 스테이징된 목록을 구성할 때 포함되지 않는 문제가 발생하였는데,
다음의 명령어로 해결할 수 있었다.

```bash
git config --global core.quotepath false
```
명령어에 대한 자세한 설명은 [링크](https://git-scm.com/docs/git-config)를 참고하자.
